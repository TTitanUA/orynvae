# 07. Режим рассказчика

## 1. Назначение

Режим рассказчика - центральная функция MVP v2.

Это интерактивный режим, где AI ведет сцену, пользователь принимает решения, а история развивается ходами. Цель режима - не заменить книгу игрой, а помочь пользователю прожить события и получить материал для художественного текста.

Режим рассказчика невозможен без доступного AI. Если AI недоступен, пользователь может открыть уже сохраненный лог сессии только для чтения, но не может продолжить, завершить, пересобрать или вручную вести сцену.

## 2. Основная идея

Процесс похож на настольную ролевую сессию по ощущению:

- есть рассказчик;
- есть персонажи;
- есть ситуация;
- пользователь принимает решения;
- мир реагирует;
- последствия сохраняются.

Но MVP не должен реализовывать полноценную RPG-систему:

- без характеристик;
- без кубиков;
- без боевого баланса;
- без классов;
- без инвентарной симуляции;
- без правил настольной игры.

## 3. Роли пользователя

Перед главой пользователь выбирает роль.

### Управляю одним персонажем

Пользователь действует от лица выбранного персонажа.

Подходит для максимального погружения.

AI должен:

- не принимать ключевые решения за этого персонажа;
- описывать ощущения и ситуацию;
- спрашивать пользователя перед важным выбором.

### Управляю несколькими персонажами

Пользователь может задавать действия нескольким героям.

Подходит для сцен с группой.

AI должен:

- уточнять, чей ход сейчас важен;
- не путать голоса персонажей;
- давать возможность переключать фокус.

### Я автор

Пользователь вмешивается сверху:

- задает направление;
- просит усилить конфликт;
- решает, что должно случиться;
- меняет тон.

AI может активнее описывать действия персонажей, но важные решения все равно лучше показывать пользователю.

### Пока не знаю

AI предлагает мягкий режим:

- больше вариантов;
- больше вопросов;
- меньше давления на выбор.

## 4. Структура хода

Один ход состоит из:

1. контекста текущей ситуации;
2. ответа AI;
3. вариантов действий;
4. действия пользователя;
5. сохранения события в лог.

### Ответ AI

Должен быть достаточно коротким, чтобы пользователь не терял контроль.

Рекомендуемый размер MVP:

- 1-4 абзаца описания;
- 2-4 варианта действия;
- возможность свободного ввода.

AI не должен генерировать целую главу без участия пользователя.

### Варианты действия

Каждый вариант должен быть осмысленным:

- действие;
- реплика;
- риск;
- осторожный путь;
- неожиданный путь.

Нельзя давать только косметические варианты.

Пример плохих вариантов:

- "Пойти налево";
- "Пойти направо";
- "Остаться".

Пример хороших вариантов:

- "Спрятать воспоминание и сделать вид, что ты ничего не нашел";
- "Открыто обвинить архивариуса";
- "Прочитать воспоминание прямо сейчас, рискуя потерять время";
- "Спросить, почему охрана пришла именно сейчас".

## 5. Типы пользовательского ввода

### Действие

Пользователь пишет, что делает персонаж.

### Реплика

Пользователь пишет, что персонаж говорит.

### Мысль или намерение

Пользователь описывает внутреннее состояние:

> Я не хочу ему верить, но делаю вид, что согласен.

### Авторская команда

Пользователь управляет сценой сверху:

> Пусть сейчас появится угроза, но не раскрывай, кто за ней стоит.

### Просьба о помощи

Пользователь может спросить:

- "Что я могу сделать?"
- "Объясни, что происходит."
- "Дай варианты реплик."
- "Сделай сцену напряженнее."
- "Замедли темп."

## 6. Панель режима рассказчика

### Центральная лента

Содержит:

- ответы рассказчика;
- действия пользователя;
- важные события;
- мягкие системные заметки.

### Панель текущей ситуации

Показывает:

- место;
- участники;
- текущая цель сцены;
- уровень опасности или напряжения;
- активные линии;
- важные факты.

### Панель ввода

Содержит:

- поле свободного ввода;
- быстрые варианты действий;
- переключатель типа ввода: действие, реплика, авторская команда;
- кнопку отправки;
- кнопку "дай варианты".

### Боковая панель памяти

Показывает только релевантное:

- персонажи сцены;
- правила мира;
- активные тайны;
- связанные линии;
- предупреждения канона.

## 7. Работа с памятью

Рассказчик использует:

- подтвержденный канон;
- черновики, связанные с текущей главой;
- выбранные линии;
- правила расширения мира;
- последние события.

Если AI хочет ввести важный новый элемент, поведение зависит от настроек:

- в свободном режиме добавляет в сцену и помечает для подтверждения после главы;
- в режиме подтверждения спрашивает;
- в строгом режиме не добавляет без запроса.

## 8. Важные ограничения AI

AI не должен:

- решать за персонажа пользователя в ключевых моментах;
- менять канон без подтверждения;
- игнорировать правила мира;
- слишком рано раскрывать тайны без причины;
- резко завершать главу без согласия пользователя;
- превращать прогноз в обязательный сюжет;
- добавлять слишком много новых сущностей за одну сцену.

AI должен:

- поддерживать темп;
- задавать выборы;
- показывать последствия;
- помнить активные линии;
- предлагать помощь, если пользователь застрял;
- давать возможность паузы.

## 9. Завершение сцены

Сцену можно завершить:

- по решению пользователя;
- после достижения цели сцены;
- после сильного поворота;
- когда накопилось достаточно материала для черновика;
- если пользователь нажал паузу.

AI может предложить завершение:

> Здесь хороший момент закончить сцену: герой получил запрещенное воспоминание, но теперь архив знает о его вмешательстве.

Пользователь выбирает:

- завершить;
- продолжить еще один ход;
- перейти к сборке черновика;
- поставить на паузу.

## 10. Пауза и восстановление

MVP должен поддерживать незавершенную сессию.

При возвращении приложение показывает:

- где остановились;
- кто участвует;
- последний ход;
- активные линии;
- кнопку продолжить;
- кнопку завершить и собрать черновик.

Если AI недоступен, кнопки продолжения и завершения заблокированы. Доступен только просмотр сохраненного состояния.

## 11. Перегенерация

В MVP нужна простая перегенерация последнего ответа AI.

Варианты:

- "попробовать иначе";
- "сделай короче";
- "сделай напряженнее";
- "меньше новых деталей";
- "ближе к канону".

Полное ветвление истории не входит в MVP.

## 12. Критерии готовности

Режим рассказчика готов для MVP, если:

- пользователь может выбрать роль;
- AI ведет сцену ходами;
- пользователь может выбирать варианты или писать свое действие;
- ходы сохраняются в лог;
- AI учитывает память и линии;
- новые важные элементы не становятся каноном автоматически;
- сессию можно поставить на паузу и продолжить;
- из сессии можно перейти к сборке черновика;
- при недоступном AI режим рассказчика не запускается и не предлагает ручной fallback.
