# 02. Пользовательский путь

## 1. Общая схема

Главный путь MVP v2:

```text
Первый запуск
-> настройка AI
-> список проектов
-> старт истории
-> анализ синопсиса
-> стартовая память
-> правила расширения мира
-> стартовая точка
-> рабочее пространство проекта
-> подготовка главы
-> режим рассказчика
-> лог сессии
-> сборка черновика
-> разбор после главы
-> обновление памяти и линий
-> прогноз следующей главы
```

Пользователь не обязан проходить путь как жесткий wizard каждый раз. Жестким должен быть только первый минимальный запуск проекта: синопсис, память, стартовая точка. Дальше приложение должно позволять возвращаться к разделам и менять решения, но только при доступном AI.

Без доступного AI приложение работает только в режиме чтения. Пользователь может открыть уже сохраненные проекты, главы, память, линии, логи, черновики и прогнозы, но не может создавать, редактировать, подтверждать, продолжать сессию или запускать новые AI-сценарии.

## 2. Первый запуск

### Цель пользователя

Понять, работает ли приложение и можно ли подключить AI.

### Система должна показать

- краткий статус backend;
- статус локального хранилища;
- есть ли настроенный AI-провайдер;
- кнопку перехода к настройкам провайдера;
- кнопку перехода к проектам, если провайдер уже настроен.

### Возможные состояния

- AI-провайдер не настроен;
- провайдер настроен, но недоступен;
- провайдер доступен;
- модель не выбрана;
- приложение работает без доступного AI только в режиме чтения.

## 3. Настройка AI

### Цель пользователя

Выбрать модель, которая будет вести историю.

### Шаги

1. Пользователь выбирает тип провайдера.
2. Заполняет base URL и API key, если нужен.
3. Нажимает "Проверить".
4. Приложение показывает статус подключения.
5. Пользователь выбирает модель.
6. Приложение сохраняет провайдера как активный.

### Важное поведение

- Если провайдер внешний, UI должен явно показать предупреждение о передаче данных.
- Если провайдер локальный, UI должен показать, что данные остаются на машине пользователя, кроме самого обращения к локальному серверу.
- Ошибки должны быть человеческими: "Не удалось подключиться к Ollama по адресу..." вместо сырого stack trace.

## 4. Список проектов

### Цель пользователя

Создать новую историю или продолжить существующую.

### Экран должен показывать

- список проектов;
- название;
- краткий синопсис или первая строка идеи;
- дату последнего изменения;
- текущую главу или последний этап;
- активного AI-провайдера;
- индикатор незавершенной сессии, если она есть;
- кнопку "Начать историю".

### Основные действия

- создать проект;
- открыть проект;
- переименовать;
- архивировать;
- удалить с подтверждением;
- перейти к настройкам AI.

Если AI недоступен, доступны только открытие и чтение проекта, а также переход к настройкам AI. Создание, переименование, архивирование и удаление должны быть заблокированы до восстановления AI, чтобы не появлялся параллельный не-AI workflow.

## 5. Старт истории

### Цель пользователя

Ввести идею обычными словами.

### Пользователь вводит

- свободный синопсис;
- необязательное рабочее название;
- необязательные заметки о жанре, тоне или запретах;
- необязательное указание, хочет ли он быть героем, управлять персонажами или наблюдать как автор.

### AI должен вернуть

- краткий пересказ;
- что в идее уже есть;
- какие вопросы остаются открытыми;
- первичные сущности;
- возможные стартовые точки;
- мягкие линии истории.

## 6. Анализ синопсиса

### Цель пользователя

Понять, как AI понял задумку, и поправить его до создания проекта.

### Экран показывает

- пересказ идеи;
- список найденных персонажей;
- список найденных мест;
- список предметов;
- список правил мира;
- список тайн;
- список потенциальных конфликтов;
- список возможных линий истории;
- уточняющие вопросы.

### Пользователь может

- принять анализ;
- отредактировать любой элемент;
- удалить лишнее;
- добавить недостающее;
- попросить AI предложить еще варианты;
- ответить на вопросы;
- продолжить без ответа на часть вопросов.

Все эти действия доступны только после успешного AI-анализа. Если AI недоступен, старт истории не продолжается.

## 7. Стартовая память

### Цель пользователя

Подтвердить стартовые элементы истории.

### Принцип

Стартовая память не должна ощущаться как тяжелая энциклопедия. Это "что мы уже знаем о мире".

### Состояния элемента

- AI-предложение;
- принято как черновик;
- подтверждено как канон;
- отклонено.

### В MVP допустимо

Не требовать заполнения подробных карточек. Достаточно короткого описания и типа элемента.

## 8. Правила расширения мира

### Цель пользователя

Решить, насколько свободно AI может добавлять новые элементы во время сессий.

### Режимы

- **Свободно в черновике.** AI может вводить новые детали в сцену, но после главы они попадают в очередь подтверждения.
- **С подтверждением.** AI спрашивает перед добавлением важного персонажа, места, предмета или правила.
- **Только по запросу.** Новые сущности появляются только если пользователь просит.
- **Смешанный режим.** Мелкие детали можно добавлять свободно, крупные требуют подтверждения.

### UI должен объяснять

- что считается мелкой деталью;
- что считается важной сущностью;
- что попадет в канон;
- что останется в черновике.

## 9. Стартовая точка

### Цель пользователя

Выбрать, где история начинается.

### AI предлагает

- 2-4 варианта первой ситуации;
- кто присутствует;
- какое напряжение есть в сцене;
- чем сцена может быть интересна;
- какие линии истории она может запустить.

### Пользователь может

- выбрать вариант;
- смешать варианты;
- написать свою стартовую точку;
- попросить другие варианты;
- перейти к проекту без запуска сцены.

## 10. Рабочее пространство проекта

### Цель пользователя

Видеть, где находится история сейчас и что можно делать дальше.

### Основные блоки

- текущий синопсис;
- последняя глава или сессия;
- кнопка "Продолжить историю";
- активные линии;
- свежие предложения в память;
- последние каноничные изменения;
- прогноз ближайших направлений.

## 11. Подготовка главы

### Цель пользователя

Перед началом интерактивной главы договориться с рассказчиком о примерном фокусе.

### Пользователь выбирает

- главу новую или продолжение незавершенной;
- роль пользователя;
- персонажа под управлением;
- активные линии;
- настроение и темп;
- свободу AI вводить новые элементы.

### AI предлагает

- краткую цель главы;
- возможные начальные ситуации;
- предупреждения о забытых линиях;
- возможные риски связности.

## 12. Режим рассказчика

### Цель пользователя

Прожить сцену, влияя на события.

### Цикл хода

1. AI описывает ситуацию.
2. Приложение показывает возможные действия.
3. Пользователь выбирает действие или пишет свое.
4. AI отвечает последствиями.
5. Система сохраняет ход в лог.
6. При необходимости AI предлагает обновить память или линию, но не прерывает сцену слишком часто.

### Пользовательские действия

- действие персонажа;
- реплика;
- внутреннее решение;
- авторское вмешательство;
- просьба к AI объяснить ситуацию;
- просьба дать варианты;
- просьба изменить тон;
- пауза;
- завершение сцены.

## 13. Сборка черновика

### Цель пользователя

Получить текст сцены или главы из интерактивной сессии.

### AI должен

- сохранить смысл решений пользователя;
- убрать технические следы игры;
- оформить диалоги;
- добавить связки;
- сохранить важные последствия;
- не добавлять новый канон без подтверждения.

### Пользователь может

- принять черновик;
- редактировать черновик при доступном AI;
- попросить переписать тон;
- сократить;
- расширить;
- сделать более литературно;
- сделать проще;
- вернуть к логу.

Если AI недоступен, пользователь может только читать уже сохраненный черновик.

## 14. Разбор после главы

### Цель пользователя

Понять, что реально изменилось.

### Приложение показывает

- резюме главы;
- список событий;
- новые сущности;
- новые факты;
- изменения персонажей;
- изменения отношений;
- продвижение линий истории;
- возможные противоречия;
- открытые вопросы;
- варианты следующего шага.

### Пользователь подтверждает

- что становится каноном;
- какие линии обновить;
- что удалить;
- что оставить как черновик;
- какие направления рассмотреть дальше.

## 15. Продолжение работы

После первой главы пользователь может:

- начать следующую главу;
- открыть память истории;
- уточнить линии;
- отредактировать черновик;
- изменить правила расширения мира;
- попросить прогноз;
- вернуться к синопсису и обновить его.

Все действия, кроме чтения уже сохраненных данных, требуют доступного AI.
