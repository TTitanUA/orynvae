# 10. AI-помощники

## 1. Назначение

В MVP v2 AI не должен ощущаться как один большой чат. Он проявляется как набор контекстных помощников в конкретных местах интерфейса:

- старт истории;
- линии истории;
- подготовка главы;
- режим рассказчика;
- варианты действий;
- сборка и правка черновика;
- разбор главы;
- прогноз;
- проверка противоречий.

Пользователь не обязан понимать внутренние `agent_key`. В UI это кнопки, панели и карточки настроек рядом с рабочим местом помощника.

Все AI-помощники недоступны без активного AI-провайдера. В этом состоянии приложение показывает ранее сохраненные результаты, но не имитирует творческие AI-действия ручными fallback-механиками.

## 2. Источник правды

Текущая конфигурация агентов задается в backend:

- `backend/app/services/project_ai_settings.py`;
- `AGENT_LABELS`;
- `ACTION_AGENT_KEYS`;
- `AGENT_PRESETS`.

API настроек проекта:

- `GET /api/projects/{project_id}/ai-settings`;
- `PATCH /api/projects/{project_id}/ai-settings`.

Глобальная модалка "Модель ассистента" управляет только проектными значениями:

- активный провайдер;
- активная модель;
- температура по умолчанию;
- Top P по умолчанию.

Настройки конкретного ассистента должны находиться в карточке этого ассистента или в карточке на странице, где он используется.

## 3. Модель настроек

У проекта есть базовые настройки генерации:

- `active_provider_id`;
- `active_model_id`;
- `default_temperature`, по умолчанию `0.7`;
- `default_top_p`, по умолчанию `0.9`.

У каждого агента можно настроить источник для температуры и Top P:

- `project` - взять значение проекта;
- `agent_default` - взять пресет агента, если он есть;
- `custom` - взять пользовательское значение агента.

Если модель не объявляет поддержку `top_p`, backend не отправляет Top P в запрос, даже если значение сохранено в настройках проекта или агента.

`reasoning_effort` не является пользовательской настройкой агента в MVP v2. Если он нужен отдельному backend-действию, он передается сервисом явно, а не через карточки настроек ассистентов.

## 5. Продуктовые архетипы

Некоторые ранние продуктовые роли остаются полезными как язык проектирования, но не являются отдельными конфигурируемыми агентами в текущей реализации.

### Проводник

Не является отдельным AI agent key. В MVP его роль выполняют:

- `next_step` в рабочем пространстве;
- контекстные ссылки;
- статичные подсказки на страницах.

### Объяснятор и библиотекарь

В MVP это статичные пояснения, help-тексты и ссылки. Они не должны становиться отдельными AI-помощниками без отдельного этапа проектирования.

### Персонажный помощник, помощник сцены и диалоговый помощник

Эти роли сейчас входят в контекстные действия:

- `start_story_interviewer`;
- `chapter_preparer`;
- `narrator`;
- `draft_fragment_editor`;
- `chapter_reviewer`.

Отдельные agent key для них не заводятся, пока нет отдельной страницы, входа, выхода и тестируемого поведения.

## 6. Размещение в интерфейсе

Глобальная модалка "Модель ассистента" не должна показывать список ассистентов и их персональные настройки. Она отвечает только за проектные defaults.

Карточки конкретных ассистентов размещаются рядом с соответствующим рабочим действием:

- `story_line_generator` - рядом с AI-предложениями и AI-правкой линий;
- `chapter_preparer` - на странице подготовки главы;
- `narrator` - в режиме рассказчика;
- `narrator_action_variants` - рядом с блоком вариантов действий;
- `draft_assembler` - рядом со сборкой черновика;
- `draft_fragment_editor` - рядом с AI-правкой фрагмента;
- `chapter_reviewer` - на странице разбора главы;
- `forecaster` - на странице прогноза;
- `contradiction_checker` - рядом с проверкой противоречий.

Настройки инструкций рассказчика остаются сессионной настройкой. Они дополняют проектные настройки агента `narrator`, но не заменяют температуру, Top P, провайдера или модель.

## 7. Технический принцип

Даже если внутри используется один AI service, продуктовые запросы разделяются по намерению и мапятся на конкретного агента через `ACTION_AGENT_KEYS`.

Текущие action types:

- `analyze_synopsis`;
- `extract_story_memory`;
- `suggest_story_lines`;
- `suggest_start_points`;
- `prepare_chapter_session`;
- `narrate_turn`;
- `suggest_turn_actions`;
- `summarize_session`;
- `extract_key_events`;
- `assemble_draft`;
- `edit_markdown_fragment`;
- `review_chapter`;
- `extract_memory_updates`;
- `update_story_lines`;
- `forecast_next`;
- `check_contradictions`.

Проектные AI-действия должны использовать `project_ai_settings.execute_project_action` или `project_ai_settings.stream_project_action_events`, чтобы provider/model/temperature/Top P резолвились централизованно.

Исключение: старт истории до создания проекта использует явные bootstrap-настройки provider/model/temperature/Top P. После подтверждения проекта эти значения сохраняются как проектные defaults.

## 8. Общие правила AI-помощников

AI-помощники должны:

- работать с текущим контекстом проекта;
- отвечать коротко, если пользователь не просит подробно;
- предлагать варианты;
- объяснять простым языком;
- не менять канон без подтверждения;
- не скрывать, что предложение является гипотезой;
- сохранять авторство пользователя.

## 9. Инварианты

- Без доступного AI-провайдера проект работает только в режиме чтения.
- Нельзя добавлять ручные creative fallback workflows для создания проекта, редактирования истории, режима рассказчика, памяти, линий, сборки черновика или прогноза.
- Канон меняется только после подтверждения пользователя.
- Markdown остается единственным persisted-форматом художественного текста.
- Debug logs, AI request logs, prompt/response dumps и frontend debug entries не хранятся в SQLite.

## 10. Критерии готовности

AI-помощники готовы для MVP, если:

- ключевые действия доступны не через один общий чат, а через контекстные кнопки и панели;
- каждый помощник имеет понятный вход и выход;
- настройки проекта отделены от настроек конкретных ассистентов;
- пользователь может принять, изменить или отклонить результат;
- AI не меняет канон сам;
- ответы написаны простым языком;
- режим рассказчика, разбор главы и прогноз работают в одном цикле главы;
- при недоступном AI помощники не запускаются, а приложение остается в режиме чтения.
