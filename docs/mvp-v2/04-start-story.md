# 04. Старт истории

## 1. Назначение

"Старт истории" - первый творческий экран MVP v2. Его задача - принять сырую идею пользователя и превратить ее в стартовую основу проекта:

- понятный синопсис;
- первичные элементы памяти истории;
- первые мягкие линии истории;
- правила расширения мира;
- стартовую точку для режима рассказчика.

Экран не должен выглядеть как анкета для писателя. Пользователь должен иметь возможность начать с одного абзаца, странного образа, сцены, конфликта или персонажа.

Старт истории невозможен без доступного AI. Если AI-провайдер не настроен или недоступен, экран должен вести к настройкам AI, а не предлагать ручное создание проекта.

## 2. Основной принцип

Старт истории не фиксирует произведение. Он создает первую карту гипотез.

Важно различать:

- **синопсис** - как пользователь сейчас понимает идею;
- **AI-пересказ** - как приложение поняло идею;
- **предложение** - возможный элемент истории;
- **черновик** - рабочая версия элемента;
- **канон** - подтвержденная пользователем правда истории;
- **гипотеза** - возможное направление, которое не является планом.

## 3. Экран ввода синопсиса

### Пользовательская задача

Написать идею без страха сделать это "неправильно".

### Основные поля

#### Рабочее название

Необязательное поле.

Если пустое, AI может предложить 3-5 вариантов названия после анализа.

#### Идея истории

Большое текстовое поле.

Минимально допустимо:

- одна фраза;
- образ;
- ситуация;
- описание героя;
- описание мира;
- конфликт;
- финальная сцена, если она уже есть.

UI должен явно говорить, что финал знать не нужно.

#### Тон или настроение

Необязательное поле или набор быстрых чипов:

- мрачное;
- теплое;
- тревожное;
- приключенческое;
- мистическое;
- романтическое;
- трагичное;
- ироничное;
- пользовательский вариант.

#### Чего избегать

Необязательное поле.

Нужно для тем, жанровых решений или элементов, которые пользователь не хочет видеть.

Примеры:

- без жестоких сцен;
- без романтической линии;
- без спасения мира;
- без открытого финала;
- не делать историю слишком комедийной.

#### Моя роль в истории

Необязательный выбор:

- хочу быть главным героем;
- хочу управлять главным героем;
- хочу управлять несколькими персонажами;
- хочу быть автором и вмешиваться сверху;
- пока не знаю.

Это не окончательный выбор. Его можно менять перед каждой главой.

### CTA

Основная кнопка:

- "Разобрать идею".

Вторичная кнопка:

- "Сначала настроить AI", если провайдер недоступен.

## 4. Подсказки на экране

Подсказки должны быть короткими и прикладными.

Примеры:

- "Можно начать с одной сцены."
- "Не нужно знать финал."
- "Если есть только герой или мир - этого достаточно."
- "AI поможет вытащить из идеи места, персонажей, тайны и первую сцену."

Нельзя перегружать экран учебным текстом. Подсказки должны быть рядом с полем, а не отдельной лекцией.

## 5. AI-анализ синопсиса

### Входные данные

- исходный синопсис;
- рабочее название;
- тон;
- ограничения пользователя;
- выбранная роль, если есть;
- язык пользователя;
- активный провайдер и модель.

### Выходные данные

AI должен вернуть структурированный результат:

- короткий пересказ идеи;
- эмоциональное ядро;
- возможный жанровый вектор;
- найденные персонажи;
- найденные места;
- найденные предметы;
- найденные группы или силы;
- найденные правила мира;
- найденные тайны;
- возможные конфликты;
- возможные линии истории;
- возможные стартовые точки;
- уточняющие вопросы;
- предупреждения о неясностях.

### Важное требование

AI не должен делать вид, что история уже решена. Формулировки должны быть мягкими:

- "возможно";
- "может быть";
- "похоже, что";
- "один из вариантов";
- "если тебе это подходит".

## 6. Экран анализа

### Верхняя часть

Показывает:

- исходную идею;
- AI-пересказ;
- кнопку "AI понял неправильно";
- кнопку "Отредактировать синопсис".

### Блок "Что уже есть в идее"

Группы:

- персонажи;
- места;
- предметы;
- правила;
- тайны;
- конфликты.

Каждый элемент показывается как карточка с действиями:

- принять;
- редактировать;
- удалить;
- сделать каноном;
- оставить как черновик.

По умолчанию все элементы должны быть предложениями или черновиками, а не каноном.

### Блок "Что стоит уточнить"

AI задает до 3 важных вопросов за раз.

Вопросы должны быть простыми:

- "Кто больше всего рискует в этой истории?"
- "Что герой не должен узнать слишком рано?"
- "Хочешь, чтобы мир был ближе к реальности или явно фантастическим?"

Пользователь может:

- ответить;
- пропустить;
- попросить варианты ответов.

### Блок "Возможные линии истории"

AI предлагает мягкие линии:

- линия героя;
- линия тайны;
- линия отношений;
- линия угрозы;
- линия темы.

Каждая линия должна иметь:

- короткое название;
- простую формулировку;
- почему это может быть интересно;
- какие элементы памяти связаны с линией.

### Блок "Возможные стартовые точки"

AI предлагает 2-4 варианта первой ситуации.

Каждая стартовая точка должна иметь:

- название;
- место;
- участники;
- напряжение;
- какой выбор может появиться у пользователя;
- какие линии запускаются.

## 7. Подтверждение стартовой памяти

После анализа пользователь переходит к экрану стартовой памяти.

Приложение должно объяснить:

> Это не финальная энциклопедия. Это то, с чего история начнет помнить себя. Все можно менять.

### Минимальный набор для создания проекта

Для MVP достаточно:

- синопсис;
- хотя бы одна стартовая точка или пользовательское подтверждение "начать без стартовой точки";
- правила расширения мира;
- активный AI-провайдер.

Не обязательно иметь:

- главного героя;
- финал;
- полный список персонажей;
- структуру глав;
- список всех локаций.

## 8. Правила расширения мира

### Назначение

Определить, насколько активно AI может придумывать новое во время режима рассказчика.

### Глобальные режимы MVP

#### Свободно в черновике

AI может добавлять новые детали, но после главы они попадают в очередь подтверждения.

Подходит, если пользователь хочет живого и неожиданного процесса.

#### С подтверждением важного

AI свободно добавляет атмосферные детали, но важные сущности требует подтвердить.

Подходит по умолчанию.

#### Только по запросу

AI не добавляет важных новых элементов без прямой просьбы.

Подходит, если пользователь хочет сильнее контролировать мир.

### Типы важных сущностей

- новый персонаж с именем;
- новая локация, куда можно вернуться;
- новый магический, технологический или социальный закон;
- важный предмет;
- новая группа или организация;
- новая тайна;
- факт, меняющий прошлое;
- факт, меняющий мотивацию персонажа.

## 9. Создание проекта

После подтверждения стартовых данных система создает:

- проект;
- исходный синопсис;
- стартовую память;
- стартовые линии истории;
- выбранные правила расширения мира;
- одну или несколько стартовых точек;
- пустой список глав;
- готовность к первой интерактивной сессии.

## 10. Ошибки и восстановление

### AI недоступен во время анализа

Показать:

- понятную ошибку;
- кнопку повторить;
- кнопку открыть настройки AI.

Нельзя создавать проект, черновик проекта или стартовую память без AI-анализа. В MVP v2 нет fallback-сценария творческой работы без AI.

### AI вернул слабой или неподходящий анализ

Пользователь должен иметь действия:

- "перегенерировать";
- "сделай проще";
- "сделай темнее";
- "дай больше вариантов";
- "не добавляй новые идеи, только извлеки из моего текста".

### Пользователь ушел со страницы

Если AI был доступен и анализ уже получен, синопсис и результаты анализа могут сохраняться как незавершенный AI-start draft. Если AI недоступен, приложение не создает проект и не сохраняет новый творческий объект без анализа.

## 11. Критерии готовности

Функция готова, если:

- пользователь может создать проект из одной идеи;
- AI анализирует идею структурированно;
- пользователь может редактировать предложения до создания проекта;
- стартовая память создается без обязательного заполнения больших форм;
- правила расширения мира сохраняются;
- стартовая точка выбрана или явно пропущена;
- пользователь после создания проекта видит понятный следующий шаг: начать первую главу.
- без доступного AI старт истории заблокирован, кроме перехода к настройкам AI.
