# 03. Карта экранов

## 1. Навигационная модель

MVP v2 должен иметь простую структуру:

```text
Главная / проекты
  -> Настройки AI
  -> Старт истории
  -> Проект
       -> Обзор
       -> Память истории
       -> Линии истории
       -> Главы
       -> Режим рассказчика
       -> Черновик главы
       -> Разбор после главы
       -> Прогноз
```

Пользователь должен всегда понимать:

- где он находится;
- какой проект открыт;
- какой AI-провайдер активен;
- есть ли несохраненные изменения;
- есть ли незавершенная сессия;
- что лучше сделать дальше.

Если AI недоступен, все экраны проекта переходят в режим чтения. Навигация и просмотр остаются доступными, но любые действия создания, изменения, подтверждения, запуска сессий и сборки черновиков блокируются.

## 2. Общая оболочка приложения

### Назначение

Дать стабильную рамку для всех экранов.

### Обязательные элементы

- название приложения;
- текущий проект, если он открыт;
- активный AI-провайдер и модель;
- индикатор внешнего провайдера;
- ссылка на настройки;
- навигация по разделам проекта;
- область системных уведомлений;
- индикатор сохранения.

### Состояния

- нет проекта;
- проект открыт;
- AI недоступен;
- AI отвечает;
- идет streaming;
- есть ошибка запроса;
- есть несохраненные изменения;
- есть незавершенная сессия.

Состояние "AI недоступен" должно быть глобальным и заметным: приложение не предлагает творческие fallback-механики и ведет пользователя к настройке AI.

## 3. Экран "Главная / проекты"

### Пользовательская задача

Выбрать существующую историю или начать новую.

### Основной контент

- список проектов;
- карточка проекта;
- кнопка "Начать историю";
- ссылка на настройки AI;
- пустое состояние для первого запуска.

### Карточка проекта

Поля:

- название;
- краткий синопсис;
- дата изменения;
- последняя глава;
- статус последней сессии;
- активный провайдер;
- количество каноничных фактов;
- количество активных линий.

### Действия

- открыть;
- продолжить;
- переименовать;
- архивировать;
- удалить;
- создать дубликат в будущем, не в MVP.

Если AI недоступен, доступно только "открыть" в режиме чтения и переход к настройкам AI. Остальные действия заблокированы.

### Пустое состояние

Текст должен быть прикладным:

> У тебя пока нет историй. Начни с одной идеи, даже если она звучит сыро.

Кнопка:

- "Начать историю".

## 4. Экран "Настройки AI"

### Пользовательская задача

Подключить модель, которая будет помогать писать и вести сцены.

### Блоки

- список провайдеров;
- форма выбранного провайдера;
- проверка подключения;
- список моделей;
- настройки streaming;
- предупреждение о приватности;
- тестовый запрос.

### Важные детали UI

- локальные провайдеры должны быть визуально отделены от внешних;
- внешний провайдер должен показывать предупреждение;
- ошибка подключения должна объяснять возможные причины;
- пользователь должен видеть, какая модель активна для текущего проекта.

## 5. Экран "Старт истории"

### Пользовательская задача

Описать идею обычными словами.

### Основные элементы

- большое поле синопсиса;
- необязательное поле названия;
- необязательные заметки о тоне;
- подсказки-примеры;
- кнопка "Разобрать идею";

### Подсказки

Подсказки должны помогать, но не давить:

- "Можно написать одну сцену, образ или конфликт."
- "Не нужно знать финал."
- "AI поможет достать из идеи героев, места и тайны."

### Ошибки

- пустой синопсис;
- AI-провайдер не выбран;
- AI недоступен;
- слишком короткий синопсис, но пользователь может продолжить, если AI доступен.

При ошибке "AI-провайдер не выбран" или "AI недоступен" нельзя продолжить создание истории. Доступны только настройка AI и возврат к чтению существующих проектов.

## 6. Экран "Анализ синопсиса"

### Пользовательская задача

Проверить, правильно ли приложение поняло идею.

### Основные блоки

- пересказ идеи;
- найденные элементы;
- возможные вопросы;
- возможные линии истории;
- возможные стартовые точки.

### Действия

- принять;
- редактировать;
- удалить элемент;
- добавить элемент;
- попросить еще варианты;
- вернуться к синопсису;
- продолжить к стартовой памяти.

### Состояния элементов

- предложено AI;
- изменено пользователем;
- принято;
- отклонено.

## 7. Экран "Стартовая память истории"

### Пользовательская задача

Собрать стартовый кодекс без ощущения большой базы данных.

### Разделы

- персонажи;
- места;
- предметы;
- группы;
- правила мира;
- тайны;
- события;
- открытые вопросы.

### Карточка элемента

Поля MVP:

- название;
- тип;
- короткое описание;
- статус;
- источник: синопсис, AI-анализ, пользователь;
- заметка "почему важно", если есть.

### Действия

- принять как черновик;
- подтвердить как канон;
- отредактировать;
- объединить похожие элементы;
- отклонить;
- добавить элемент при доступном AI.

В режиме чтения эти действия недоступны.

## 8. Экран "Правила расширения мира"

### Пользовательская задача

Решить, насколько свободно рассказчик может добавлять новое.

### Основные варианты

- свободно в черновике;
- с подтверждением;
- только по запросу;
- смешанный режим.

### Детализация

Для каждого типа сущности можно выбрать отдельное поведение:

- персонажи;
- места;
- предметы;
- правила мира;
- группы;
- тайны;
- события.

В MVP можно дать общий режим и расширенные настройки под ним.

## 9. Экран "Выбор стартовой точки"

### Пользовательская задача

Выбрать первую ситуацию для режима рассказчика.

### Карточка стартовой точки

Поля:

- название сцены;
- место;
- участники;
- начальное напряжение;
- что уже известно;
- что скрыто;
- какие линии может запустить;
- насколько активна роль пользователя.

### Действия

- выбрать;
- отредактировать;
- смешать варианты;
- попросить еще;
- написать свою.

## 10. Экран "Обзор проекта"

### Пользовательская задача

Понять текущее состояние истории и продолжить.

### Основные блоки

- текущий синопсис;
- кнопка "Продолжить историю";
- незавершенная сессия, если есть;
- последняя глава;
- активные линии;
- свежие предложения в память;
- открытые вопросы;
- прогноз ближайшего развития;
- быстрые переходы.

### Важное поведение

Если есть незавершенная сессия, главный CTA должен предлагать продолжить ее, а не начать новую.

## 11. Экран "Память истории"

### Пользовательская задача

Посмотреть и управлять тем, что история уже знает.

### Представления

- все элементы;
- по типам;
- только канон;
- очередь предложений;
- возможные противоречия.

### Действия

- добавить элемент;
- редактировать;
- подтвердить;
- отклонить;
- пометить как устаревшее;
- связать с главой;
- связать с линией;
- найти использование.

В режиме чтения доступен только просмотр и поиск.

## 12. Экран "Линии истории"

### Пользовательская задача

Видеть долгие нити истории и выбирать, что продвигать дальше.

### Основные блоки

- активные линии;
- линии без внимания;
- завершенные линии;
- предложения AI;
- связь с главами.

### Карточка линии

Поля:

- название;
- тип;
- текущая формулировка;
- почему важна;
- последняя глава, где продвигалась;
- текущее состояние;
- возможный следующий шаг;
- связанные персонажи и факты.

## 13. Экран "Подготовка главы"

### Пользовательская задача

Договориться с рассказчиком о следующей сессии.

### Блоки

- номер или название главы;
- цель главы простым языком;
- роль пользователя;
- управляемые персонажи;
- активные линии;
- темп;
- тон;
- свобода AI;
- стартовая ситуация.

### Результат

После подтверждения создается chapter session и открывается режим рассказчика.

Если AI недоступен, подготовка главы не может быть подтверждена.

## 14. Экран "Режим рассказчика"

### Пользовательская задача

Играть сцену и влиять на историю.

### Макет

- центральная лента сцены;
- панель текущей ситуации;
- поле действия пользователя;
- быстрые варианты действий;
- панель персонажа или роли пользователя;
- боковая память: важные факты, линии, предупреждения;
- кнопки паузы, завершения сцены, сборки черновика.

### Виды пользовательского ввода

- действие;
- реплика;
- мысль персонажа;
- авторская команда;
- просьба объяснить;
- просьба дать варианты;
- просьба изменить тон.

### Важные ограничения

- AI не должен уходить далеко без хода пользователя;
- важные новые сущности не становятся каноном автоматически;
- пользователь может откатить последний AI-ответ в рамках MVP как "перегенерировать", но полноценное дерево веток не требуется.

## 15. Экран "Лог сессии"

### Пользовательская задача

Понять, что произошло в интерактивной главе.

### Лог должен показывать

- ходы AI;
- ходы пользователя;
- ключевые события;
- важные решения;
- созданные сущности;
- потенциальные факты;
- связанные линии.

### Действия

- пометить событие важным;
- исключить событие из черновика;
- отредактировать формулировку;
- перейти к сборке черновика.

Если AI недоступен, лог доступен только для чтения.

## 16. Экран "Сборка черновика"

### Пользовательская задача

Превратить сессию в художественный markdown.

### Макет

- слева лог или список ключевых событий;
- справа черновик главы;
- сверху режим сборки;
- снизу AI-инструменты редактуры.

### Режимы сборки

- близко к событиям;
- более литературно;
- короче;
- подробнее;
- с фокусом на диалог;
- с фокусом на атмосферу.

## 17. Экран "Разбор после главы"

### Пользовательская задача

Подтвердить изменения и понять последствия.

### Блоки

- резюме главы;
- новые факты;
- изменения персонажей;
- изменения отношений;
- новые сущности;
- обновления линий;
- возможные противоречия;
- открытые вопросы.

### Действия

- подтвердить в канон;
- оставить черновиком;
- отклонить;
- отредактировать;
- создать новую линию;
- обновить существующую линию.

Если AI недоступен, разбор доступен только для чтения.

## 18. Экран "Прогноз"

### Пользовательская задача

Посмотреть возможные направления без фиксации плана.

### Прогноз должен показывать

- 2-4 варианта следующей главы;
- какие линии они продвигают;
- возможные риски;
- вероятные последствия;
- возможные будущие финалы как гипотезы;
- предупреждение, что это не план.

### Действия

- выбрать направление как ориентир;
- отложить;
- попросить другие варианты;
- перейти к подготовке главы.

Если AI недоступен, прогноз доступен только для чтения.
